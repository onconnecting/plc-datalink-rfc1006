## [Unreleased] — 2026-05-15

### Security
- Externalize CouchDB admin credentials and container registry settings via `.env` file. The compose stacks, `database/config/init-db.sh`, and the manual CouchDB test scripts under `database/test/` no longer carry hardcoded `admin:password` strings.

- `.claude/hooks/run-tests-on-edit.sh` — PostToolUse dispatch script that runs the layer-specific test suite in its dedicated container (`backend-test`, `frontend-test`, `database-test`) after every Edit/Write under `backend/src/**`, `frontend/src/**`, or `database/config/**`. Honors the project rule that tests run in containers, not on the host; skips quietly if the matching image is not built so first-time use is not blocked by image builds. Provides the runtime-enforcement half of [ADR-0007](architecture/decisions/ADR-0007-post-tool-use-hooks-test-auto-run.md), complementing the skill-instruction half already in `.claude/skills/03-backend|04-frontend|05-database`. Wiring requires a `hooks.PostToolUse` block in `.claude/settings.json` that points at this script — written by hand because the harness's auto-mode classifier (correctly) refuses to let Claude install auto-executing hooks on itself.
- `.claude/settings.json` permissions for `Bash(pytest:*)`, `Bash(cd backend && pytest:*)`, `Bash(cd frontend && npm test:*)`, `Bash(cd database && pytest:*)`, and `Edit(.claude/hooks/**)` per [ADR-0007](architecture/decisions/ADR-0007-post-tool-use-hooks-test-auto-run.md) §3.
- Backend test suite under `backend/test/scripts/` per [ADR-0008](architecture/decisions/ADR-0008-testcontainers-couchdb-integration-tests.md): pytest layout split into `unit/` (model, services, routes — all dependencies mocked) and `integration/` (Telegraf renderer against the full ZKS tag set, CouchDB roundtrip via testcontainers, S7 smoke against the ZKS machine mock). Every test under `integration/` is auto-tagged with the `integration` marker so the unit suite can be run in isolation (`pytest -m "not integration"`). Builds via `backend/Dockerfile.test` (deps from `pyproject.toml[dev,e2e]`), runs via `docker compose -f dc-plc-datalink-rfc1006-test.yml run --rm backend-test`. `backend/Dockerfile.test.dockerignore` keeps the test image lean by excluding `.venv/`, `__pycache__/`, and build artefacts. `backend/README.md` documents the unit / integration / E2E flow.
- `pyproject.toml` `e2e` extras group (`paho-mqtt`, `python-snap7`, `testcontainers>=4`) and `dev` additions (`pytest-mock`, `requests-mock`); pytest `markers = ["integration: ..."]` registered to silence `PytestUnknownMarkWarning` from the auto-applied marker.
- End-to-end stack `dc-plc-datalink-rfc1006-e2e.yml` (no host ports, no shared volumes, separate `plc-datalink-rfc1006-e2e-net`) plus `backend/test/scripts/integration/test_e2e_zks_mqtt.py`: walks REST → CouchDB → supervisord/Telegraf → ZKS S7 → MQTT against a production-image backend. Wires four containers — `couchdb-e2e` (real database image), `mosquitto-e2e` (real broker, anonymous internal listener), `backend-e2e` (production backend image with Telegraf), `backend-e2e-runner` (the `backend-test` image, runs only the E2E pytest). The runner reaches the host-bound ZKS mock via `host.docker.internal:host-gateway`; session-start fixtures probe ZKS, the backend, and Mosquitto and skip cleanly when ZKS is unreachable.
- Database test suite under `database/test/python/` per [ADR-0008](architecture/decisions/ADR-0008-testcontainers-couchdb-integration-tests.md): 35 unit tests (`init-db.sh` content, `local.ini` CORS posture, `Dockerfile` couchdb:3.x pin, ZKS layout parse, S7→RFC1006 address translation per README spec) plus 10 integration tests against a session-scoped `couchdb:3.3.3` testcontainer with per-test ephemeral databases — covering bootstrap (`_users`, `_replicator`, `_global_changes`, `datalink`), admin auth (401 anonymous), and full ZKS-machine-doc CRUD (round-trip of all 128 PLC tags, `_rev` conflict → 409, `_rev`-aware update, `_all_docs?include_docs=true`, delete with `?rev=`). Run via `docker compose -f dc-plc-datalink-rfc1006-test.yml run --rm database-test`. Helper `database/test/python/zks_layout.py` builds the `PlcDatalinkRFC1006`-shaped document directly from the ZKS YAML, so the integration suite stays in sync with the canonical machine description.
- `qa/evidence/zks-database-tests-qa-2026-05-15.md` — QA evidence for the database test suite (10/10 acceptance criteria, three Low findings, all addressed in this changelog entry).
- Frontend test baseline per [ADR-0006](architecture/decisions/ADR-0006-jest-for-frontend-tests.md): Jest + `jest-preset-angular`, split into `frontend/test/unit/` (9 specs, 66 tests — `AppComponent`, `HeaderComponent`, `ConfigurationOverviewComponent`, `CreateConfigurationComponent`, `PlcStatesComponent` with mocked services; `ConfigService`, `MachineService` via `HttpClientTesting`; pure-function `formatBackendError` and `plc-validators` covering every PLC type from the README — `X B C W DW I DI R DT S`) and `frontend/test/integration/` (2 specs, 6 tests — raw `fetch` against the real Flask backend exercising `/config/*` CRUD round-trip incl. the double-JSON-encoded `/config/create` quirk, and `/machine/*` list endpoints + optional ZKS-Mock-touching `/machine/start` flow). `angular.json` schematics `skipTests` flipped to `false` for component/directive/pipe so generated artefacts ship with a spec by default. New compose stack `dc-plc-datalink-rfc1006-test.yml` orchestrates `frontend-test`, `frontend-test-int`, `backend-int` (real Flask), and `database-int` (real CouchDB) on the `plc-datalink-rfc1006-test-network` bridge; backend reaches the external ZKS-Mock at `host.docker.internal:102` via `extra_hosts: host-gateway`. ZKS-touching integration spec auto-skips when ZKS is unreachable (TCP probe on port 102) so the suite passes without an external dependency.

### Changed
- **Frontend complete rewrite** (per [ADR-0003](architecture/decisions/ADR-0003-frontend-ui-foundation-angular-cdk.md) + [ADR-0004](architecture/decisions/ADR-0004-frontend-greenfield-migration-strategy.md)): `frontend/` is now an Angular 19 LTS standalone-components project with Signals, OnPush, the new control-flow (`@if`/`@for`), typed reactive forms, `inject()` DI, and `takeUntilDestroyed`. UI primitives (`oc-button`, `oc-input`, `oc-field`, `oc-card`, `oc-status-pill`, `oc-toast` + `ToastService`, `oc-dialog` + `DialogService`) are built directly on `@angular/cdk` — no Bootstrap, Material, PrimeNG, or Tailwind. Design tokens (`src/styles/_tokens.css`) are the single colour/spacing/typography source; `stylelint` enforces no raw hex outside the token file. UI tonality is German per the onconnecting CI manual, no emojis, no exclamation chains; PLC addresses, IPs, ports, and topics render in Consolas. Routes (`/plc-states`, `/configuration-overview`, `/create-configuration`) and backend contract (relative `/config/*` and `/machine/*` paths) are preserved 1:1; the legacy `/api/*` prefix is dropped. Build budgets: 374 kB initial / 101 kB transfer. ESLint + Stylelint clean.
- Greenfield rewrite landed via the directory swap ADR-0004 prescribes: `frontend-next/` → `frontend/`, legacy `frontend/` deleted (history preserved in git). `dc-plc-datalink-rfc1006-dev.yml` requires no change — `build.context: ./frontend` keeps pointing at the same path with new content.
- `dc-plc-datalink-rfc1006-local.yml` and `dc-plc-datalink-rfc1006-acr.yml` now read CouchDB credentials via `${COUCHDB_USER:-admin}` / `${COUCHDB_PASSWORD:-password}` and inject `DATABASE_USER_NAME` / `DATABASE_SECRET_KEY` into the backend container.
- Rename `backend/src/init.py` → `backend/src/app.py`; the empty `backend/src/__init__.py` is kept. The Gunicorn target in `backend/config/supervisord.conf` is updated to `src.app:app`. See ADR-0001 for the rationale.
- Rename `backend/config/env` → `backend/config/env.example` to mark it explicitly as a template; the Dockerfile copies it to `/app/.env` at build time (unchanged runtime behavior).
- Reorganize `backend/test/`: shell-curl scripts moved into `backend/test/curl/`, the Python dev helper into `backend/test/scripts/`. No script logic changed.
- The ACR compose file references images via `${ACR_REGISTRY:-onconnecting.azurecr.io}/...:${IMAGE_TAG:-latest}`.
- `backend/config/env` no longer carries the CouchDB credentials; they are supplied by the compose `environment:` section.
- `database/test/couch-cmd.sh` and `database/test/devBoard1Rest.sh` fail fast if `COUCHDB_USER` / `COUCHDB_PASSWORD` are not exported.
- Rename the three Dockerfiles to the canonical `Dockerfile` (`backend/Dockerfile`, `frontend/Dockerfile`, `database/Dockerfile`). The previous `dockerfile-plc-datalink-rfc1006-{backend,frontend,database}` names were lowercase, redundantly carried the repo prefix, lost IDE/GitHub-linguist syntax detection, and blocked Dependabot's `docker` ecosystem (which only matches `Dockerfile` / `Dockerfile.*`). `dc-plc-datalink-rfc1006-dev.yml` drops the explicit `dockerfile:` keys (the default is now correct), `.github/workflows/docker-image.yml` drops the `--file` argument from the three `az acr build` calls, and `.github/dependabot.yml` now enables `docker` ecosystem updates for `/backend`, `/frontend`, and `/database`. Internal references in `.claude/rules/`, `.claude/skills/`, `backend/README.md`, and `database/README.md` updated. No behavior change — the ACR compose stack is unaffected (image-only, no build).

### Removed
- Tracking of the `.claude/` directory and `doc/design/onconnecting-ci/` from version control via `.gitignore`.
- `dc-plc-datalink-rfc1006-local.yml` — superseded by `dc-plc-datalink-rfc1006-dev.yml`, which runs the same three services but goes through the local registry (build → push → pull → up) so the DEV loop matches the PROD pull mechanics.
- Top-level `Makefile` — the stack is bedient directly with `docker compose` calls; backend lint/format runs as `ruff check` / `ruff format` from `backend/`. README documents the new commands.
- `backend/test/scripts/plc_datalink_rfc1006_linux.py` — orphan legacy Flask-app duplicate, not imported by any test (the suites under `backend/test/scripts/unit/` and `backend/test/scripts/integration/` import `src.plc_datalink_rfc1006_model` directly) and not referenced by Dockerfile, compose, or any shell script.
- Tracking of `backend/.venv/` (1130 files) from version control. The directory was committed before `.gitignore` excluded `.venv/`; the on-disk venv is untouched and `.gitignore` already prevents reintroduction. Untracked via `git rm -r --cached backend/.venv/`.

### Fixed
- `docs/features/test-strategy/scope.md` — correct the STRING-address rows in the *Repräsentativer ZKS-Tag-Subset* table (`DB1.S20.20` → `DB1.S38.20` for `Machine.LastError`, `DB4.S32.32` → `DB4.S0.32` for `Part.Serial`) and the format-syntax footnote (`S<max>.<len>` → `S<offset>.<max>`, matching the README's `DB2000.S30.13` example). Surfaced by QA against the database test suite; would have caused the future `/03-backend` E2E runner to point Telegraf at the wrong byte offsets.
- `architecture/decisions/ADR-0008-testcontainers-couchdb-integration-tests.md` — appended an *Update 2026-05-15 — Implementation notes* section: the `[couchdb]` pip extra was removed from `testcontainers` 4.x (pip warns when it is requested), so the suite installs plain `testcontainers>=4` and instantiates `DockerContainer("couchdb:3.3.3")` directly. The dependency lives in `database/Dockerfile.test`, not `backend/pyproject.toml`. The core decision (testcontainers + couchdb:3 per session, ephemeral DB per test) is unchanged.
- `database/test/python/conftest.py` — migrate from the deprecated `wait_for_logs(...)` call to the structured `LogMessageWaitStrategy` (imported from `testcontainers.core.wait_strategies`), applied via the container's `.waiting_for(...)` builder. Removes the `DeprecationWarning` from the test output and tracks the wait-strategy API forward to the next testcontainers major.
- `backend/test/scripts/integration/conftest.py` — mirror the database conftest's switch to `LogMessageWaitStrategy.with_startup_timeout(60)` for the CouchDB testcontainer; drops the matching `DeprecationWarning` from the backend integration run.
- `dc-plc-datalink-rfc1006-test.yml` (`frontend-test-int`) and `dc-plc-datalink-rfc1006-e2e.yml` (`backend-e2e-runner`) — default `ZKS_S7_HOST` from `host.docker.internal` to `192.168.0.180`, the canonical ZKS mock address declared in [`docs/machines-db-layout/zks-machine-mock/db-layout.yaml`](docs/machines-db-layout/zks-machine-mock/db-layout.yaml). Surfaced by QA: the previous default assumed the mock co-located with the test runner, while the mock actually lives on a separate LAN host. Both compose files now use `${ZKS_S7_HOST:-192.168.0.180}` so developers running a local mock can override via `.env`.
- `.gitignore` now also excludes `backend/.venv/`, `__pycache__/`, `*.pyo`, `*.egg-info/`, `.pytest_cache/`, `.coverage`, `htmlcov/`, `node_modules/`, `frontend/dist/`, `.angular/`, and `*.swp` / `*.swo` swap files — previously these could have been accidentally committed.
- Bump the pinned SHA256 of the InfluxData GPG key in `backend/dockerfile-plc-datalink-rfc1006-backend` to `40557e26…4d5dac` (the value currently served at `repos.influxdata.com/influxdata-archive.key`). The previous pin (`943666…7515`) no longer matched after an upstream key rotation, blocking the backend image build at step 4 of 20.

### Notes
- The newly enabled lint workflow (Ruff for backend, Prettier for frontend) will likely fail on the first push because existing files were written before either tool was configured. Run `ruff format src test` inside `backend/` and `npx prettier --write "src/**/*.{ts,js,html,css,scss}"` inside `frontend/` once to establish a baseline.
- Pre-commit hooks only activate after `pre-commit install` is run in the working tree.
- The DEV registry is **insecure** (HTTP). Every host that pushes or pulls must list `192.168.0.121:5000` under `insecure-registries` in `/etc/docker/daemon.json` and reload the daemon (`systemctl restart docker`). README contains the one-liner. Do not reuse this registry for PROD.
